# Agent Dashboard 可视化界面设计

> **Date:** 2026-03-14
> **Status:** 已确认

---

## 1. 项目概述

为 EmbodiedAgentsSys 构建一个 Web 可视化控制面板，用于实时监控和交互。

### 目标用途

- 实时监控与调试
- 用户交互界面
- 演示展示

### 部署环境

- 本地开发机
- 机器人端
- 云端/远程服务器（灵活部署）

## 4. 模块详细设计

### 4.1 顶部状态栏

显示当前系统的关键状态信息：

| 状态项 | 图标 | 说明 |
|--------|------|------|
| **连接状态** | 🟢/🔴 | WebSocket/ROS 连接状态 |
| **机器人状态** | 🤖 | Idle / Working / Error 等 |
| **当前模型** | 🧠 | 当前使用的大模型（如 Qwen3L） |
| **活跃 Skills** | ⚡ | 当前加载的技能数量 |
| **设置按钮** | ⚙️ | 跳转设置页面 |

### 4.2 侧边栏

导航菜单，包含以下选项：

- 📝 对话
- 📷 相机
- 🎯 检测
- ⚙️ 设置

### 4.3 主内容区域

根据侧边栏选择显示对应内容：

#### 4.3.1 对话面板

- **历史对话列表**：显示用户输入和系统回复
- **输入框**：输入新指令/问题
- **发送按钮**：提交指令
- **清空按钮**：清空对话历史

**数据流：**
- 用户输入 → HTTP POST → 后端 → VLA/LLM 处理
- 响应结果 → WebSocket 推送 → 前端显示
- 历史记录保存在本地（localStorage）

#### 4.3.2 相机画面

- **实时视频显示区域**
- **截图按钮**：捕获当前帧
- **画面质量选择**：低/中/高

**数据流：**
- 视频流 → WebSocket 推送 Base64 帧 / MJPEG 流 → 前端渲染

#### 4.3.3 检测结果

- **检测目标列表**：显示识别到的物体
- **分割结果可视化**：在图像上叠加分割掩码
- **目标详情**：类别、置信度、位置

**数据流：**
- 检测结果 → WebSocket 推送 JSON → 前端渲染

#### 4.3.4 设置面板

| 设置项 | 类型 | 说明 |
|--------|------|------|
| **语言** | 下拉选择 | 中文/English |
| **界面风格** | 下拉选择 | 浅色/深色/自动 |
| **大模型选择** | 下拉选择 | 选择使用的 VLA/LLM 模型 |
| **WebSocket 地址** | 输入框 | 实时数据连接地址 |
| **API 地址** | 输入框 | HTTP API 地址 |
| **刷新频率** | 滑块 | 相机画面/状态更新频率 |

**数据存储：**
- 设置保存在 localStorage

## 5. 通信协议

### 5.1 HTTP API

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/healthz` | 健康检查 |
| POST | `/api/chat` | 发送聊天消息 |
| GET | `/api/camera/frame` | 获取当前帧 |
| GET | `/api/detection/result` | 获取检测结果 |
| GET | `/api/settings` | 获取设置 |
| PUT | `/api/settings` | 更新设置 |

### 5.2 WebSocket

| 事件 | 方向 | 说明 |
|------|------|------|
| `camera_frame` | Server → Client | 推送相机画面 |
| `detection_result` | Server → Client | 推送检测结果 |
| `robot_status` | Server → Client | 推送机器人状态 |
| `chat_message` | Server → Client | 推送对话响应 |
| `send_message` | Client → Server | 发送聊天消息（可选） |
