# 工业 Agent 设计准则

> 工业 Agent 不是聊天机器人，而是关乎安全的系统。  
> 核心目标：**放弃追求"聪明"，专注打造绝对安全可靠的工业系统。**

---

## P1 安全优先层

- **零错误容忍**：错误 = 事故，不存在"可接受的失误"
- **不确定即禁止**：无法确认安全性的操作，一律拒绝执行
- 安全性优先级高于效率、高于功能、高于用户体验

---

## P2 可控接管层

- **人工随时可介入**：任何时刻操作员均可接管系统控制权
- **主动上报异常**：系统检测到异常须立即告警，不得静默失败
- **权限分级管理**：按角色划分操作权限，避免越权执行

---

## P3 决策架构层

大模型不得直接控制设备，必须采用三层分离架构：

| 层级 | 职责 | 说明 |
|------|------|------|
| **大模型（LLM）** | 意图理解 + 方案提案 | 只输出建议，不下达指令 |
| **规则校验系统** | 白名单审核 + 边界检查 | 拦截高危指令，校验合规性 |
| **执行层** | 设备控制 + 边缘计算 | 具备离线降级能力 |

> LLM 提案 → 规则系统校验 → 执行层执行，缺一不可。

---

## P4 闭环设计层

所有指令必须经过完整闭环，不得仅发出指令而不确认结果：

```
感知（Sense）→ 决策（Decide）→ 执行（Execute）→ 反馈确认（Confirm）
                                                        │
                                            ◄───────────┘（未确认 = 失败）
```

- 指令执行后**必须收到确认结果**，否则视为执行失败并触发告警
- 禁止"发出即完成"的单向指令模式

---

## P5 可靠性层

### 毫秒级响应
- 实时控制链路不可引入不可控延迟
- 大模型推理不得阻塞关键控制路径

### 7×24 稳定运行
- 边缘节点须具备**离线降级**能力，防止云端断连导致停工
- 制定明确的降级方案（Fallback），确保核心功能持续可用

### 故障隔离
- 单点失效不得级联扩散
- 各模块独立部署，具备自愈或安全停机能力

---

## P6 安全机制层

### 白名单制度
- 所有可执行指令须预先在白名单中登记
- 未授权指令**一律拒绝**，不做推断，不做例外

### 二次确认
- 高危操作（停机、参数变更、阀门开关等）强制人工审批后方可执行
- 系统不得自动绕过二次确认流程

### 完整审计日志
- 全链路操作记录，包括：指令来源、校验结果、执行时间、反馈状态
- 日志不可篡改，支持事后追溯与合规审查

---

## 七大铁律速查

| # | 铁律 | 核心要求 |
|---|------|----------|
| 1 | 不能错 | 错误 = 事故，零容忍 |
| 2 | 必须可控 | 人工随时可接管 |
| 3 | 毫秒级响应 | 实时控制不可延迟 |
| 4 | 7×24 稳定 | 边缘节点 + 降级方案 |
| 5 | 闭环设计 | 感知→决策→执行→反馈确认 |
| 6 | LLM 不直控设备 | 大模型提案，规则系统校验，执行层执行 |
| 7 | 安全机制拉满 | 白名单 + 二次确认 + 审计日志 |

---

*工业 Agent 的价值不在于它有多聪明，而在于它有多可靠。*
