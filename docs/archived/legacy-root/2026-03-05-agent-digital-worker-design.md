# Agent数字员工设计方案

> **文档版本**：v1.0  
> **创建日期**：2026年3月5日  
> **项目**：AIR Robotics - 工业机械臂Agent数字员工

---

## 1. 项目背景与目标

### 1.1 项目背景

传统工业机械臂在应对现代制造业"小批量、多品种、高频迭代"需求时，暴露出一系列刚性瓶颈：
- 调试周期长，依赖工程师人工示教
- 柔性不足，"一机一用"难以适应多品种生产
- 操作门槛高，需要专业编程能力

### 1.2 项目目标

打造一款Agent数字员工产品，让机械臂能够：
- **看懂**：通过视觉感知理解环境和工件
- **思考**：理解人类指令并自动规划执行路径
- **学会**：通过一次示教自主掌握新技能
- **执行**：完成上料、装配、搬运等任务

### 1.3 目标场景

| 场景 | 描述 | 典型指令 |
|-----|------|---------|
| 场景1：上料/预组装 | 将物料从料框搬运到产线 | "把这箱托盘搬到产线" |
| 场景2：柔性装配 | 多品种小批量装配任务 | "把端板装配到托盘上" |
| 场景3：语音示教 | 通过语音控制机械臂动作 | "把零件拿到拍照位置" |

## 2. 整体架构设计

### 2.1 层级式Agent架构

```
┌─────────────────────────────────────────────────────────┐
│                    用户层（语音/界面）                    │
│        语音输入 | 示教手柄 | Web界面 | API调用            │
└────────────────────────┬────────────────────────────────┘
                         │
┌────────────────────────▼────────────────────────────────┐
│              任务理解层（Task Understanding）              │
│        语音/文本语义解析 → 意图识别 → 参数提取            │
└────────────────┬────────────────────────────────────────┘
                 │
┌────────────────▼────────────────────────────────────────┐
│              任务规划层（Task Planning Agent）            │
│        任务拆解 | 冲突检测 | 路径规划 | 执行序列生成      │
└────────────────┬────────────────────────────────────────┘
                 │
┌────────────────▼────────────────────────────────────────┐
│              技能执行层（Skill Execution Agent）          │
│        ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐   │
│        │视觉感知 │ │运动控制 │ │力控调节 │ │夹爪管理 │   │
│        └─────────┘ └─────────┘ └─────────┘ └─────────┘   │
└────────────────┬────────────────────────────────────────┘
                 │
┌────────────────▼────────────────────────────────────────┐
│              学习进化层（Learning Agent）                │

```

### 2.2 各层职责

| 层级 | 核心职责 | 关键组件 |
|-----|---------|---------|
| 用户层 | 多模态交互入口 | 语音ASR、Web UI、手柄/API |
| 任务理解层 | 指令解析与意图识别 | 语义理解、参数提取 |
| 任务规划层 | 任务拆解与运动规划 | LLM规划器、冲突检测、路径优化 |
| 技能执行层 | 底层控制与感知执行 | 视觉模块、运动控制器、力控模块 |
| 学习进化层 | 技能沉淀与持续优化 | 示教学习、参数调优、知识库 |

## 3. 核心能力设计

### 3.1 视觉识别能力

**功能**：通过多目视觉/3D重建感知工件位置、姿态、类型

**技术实现**：
- 3D Gaussian Splatting语义场重建
- Next-Best-View自主探索扫描
- 语义分割识别"抓取面"、"避障区"

**输出**：
- 工件3D位姿（xyz + rx ry rz）
- 工件类型/品类识别
- 抓取点推荐

### 3.2 任务规划能力

**功能**：将"把A零件装到B位置"这样的指令自动拆解为可执行的动作序列

**技术实现**：
- 基于LLM的任务拆解（如GPT-4/Claude）
- RT-2/GR-2风格的视觉-语言-动作模型
- 运动规划（Motion Planning）

**示例**：
```
用户指令："把端板装配到托盘上"

→ 任务拆解：
1. 视觉定位"端板"当前位置
2. 视觉定位"托盘"目标位置
3. 规划抓取路径（避开障碍）
4. 抓取端板
5. 规划搬运路径
6. 移动到托盘上方
7. 力控插入（检测对位）
8. 确认装配完成
```

### 3.3 自主学习能力

**功能**：通过一次示教/遥操，自动学会新技能，后续只改参数就能迁移到新产品

**技术实现**：
- 遥操示教（VR/AR + 数字孪生）
- 技能参数化（输入参数、执行逻辑、质量判定标准）
- LoRA微调分钟级更新模型

**Skill定义规范**：
```yaml
SkillName: "托盘上料"
InputParams:
  - source_position: 料框位置
  - target_position: 产线位置
  - workpiece_type: 工件类型
ExecutionLogic:
  - 视觉识别工件
  - 规划抓取路径
  - 执行抓取
  - 移动到目标位
  - 放置
QualityCriteria:
  - 成功率 > 95%
  -  cycle_time < 30s
```

## 4. 场景详细设计

### 4.1 场景1：上料/预组装

**用户指令**："把这箱托盘搬到产线"

**Agent执行流程**：

```
1. 视觉识别
   ├── 识别料框位置
   ├── 识别箱中托盘数量/状态
   └── 生成抓取策略

2. 任务规划
   ├── 拆解：逐件搬运
   ├── 规划每件抓取点
   └── 规划搬运路径

3. 技能执行
   ├── 机械臂移动到抓取位
   ├── 夹爪抓取托盘
   ├── 搬运到产线上料位
   ├── 放置
   └── 状态检测（是否成功）

4. 循环执行
   └── 重复直至料框搬空

5. 学习进化
   ├── 记录本次执行数据
   └── 优化下次抓取策略
```

**技术指标**：
- 视觉识别准确率：> 95%
- 抓取成功率：> 93%
- 单件cycle time：< 30s

### 4.2 场景2：柔性装配

**用户指令**："把端板装配到托盘上"

**Agent执行流程**：

```
1. 视觉定位
   ├── 识别托盘基准位
   ├── 识别端板当前位置
   └── 生成装配坐标系

2. 任务规划
   ├── 装配序列规划
   ├── 力控参数规划
   └── 异常应对预案

3. 技能执行（力控装配）
   ├── 移动到端板上方
   ├── 视觉精确对位
   ├── 缓慢下降接触
   ├── 力控插入（阻抗控制）
   ├── 位置校验
   └── 确认完成

4. 异常处理
   ├── 插入阻力过大 → 暂停 + 视觉重检
   ├── 位置偏移 → 重新规划
   └── 装配失败 → 报警 + 人工介入

5. 学习进化
   ├── 记录力控参数
   └── 优化装配策略
```

**技术指标**：
- 装配精度：±0.5mm（当前）→ ±0.2mm（下一阶段）
- 装配成功率：> 88%
- 力控响应时间：< 10ms

### 4.3 场景3：语音示教

**用户指令**：
- 基础运动："向前20cm"、"向上5cm"
- 复合动作："把夹爪打开"、"移动到料框上方"
- 任务级："把零件拿到拍照位置"

**语音示教架构**：

```
用户语音输入
    │
    ▼
┌─────────────────────────────────────────┐
│  语音理解模块（ASR + 语义解析）           │
│  输入："把零件拿到拍照位置"                │
│  输出：                                    │
│  {                                        │
│    intent: "移动物体",                    │
│    object: "零件",                        │
│    target: "拍照位置",                    │
│    gripper: "需要抓取"                   │
│  }                                        │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  参数化Skill生成                          │
│  根据语义解析结果，动态生成执行Skill：     │
│  1. 视觉定位"零件"位置                    │
│  2. 夹爪→闭合（抓取）                    │
│  3. 移动到"拍照位置"                      │
│  4. 夹爪→打开（释放）                    │
└────────────────┬────────────────────────┘
                 │
                 ▼
┌─────────────────────────────────────────┐
│  执行与反馈                               │
│  - 250Hz控制循环执行                      │
│  - 实时位置反馈                           │
│  - 语音播报状态："开始移动...已完成"      │
└─────────────────────────────────────────┘
```

**语音指令覆盖级别**：

| 级别 | 示例指令 | 技术难度 |
|-----|---------|---------|
| L1 基础运动 | "向前20cm"、"向上5cm" | 低 |
| L2 复合动作 | "把夹爪打开"、"移动到料框上方" | 中 |
| L3 任务级 | "把零件拿到拍照位置" | 高 |

**技术指标**：
- ASR识别准确率：> 95%
- 语义理解准确率：> 90%
- 执行响应时间：< 500ms

## 5. 技术选型

### 5.1 核心技术组件

| 模块 | 技术选型 | 说明 |
|-----|---------|------|
| 语音ASR | 阿里云ASR / 科大讯飞 | 支持工业噪声环境 |
| 语义理解 | LLM（GPT-4/Claude/国产大模型） | 意图识别 + 参数提取 |
| 视觉感知 | 3DGS + 多目视觉 | 语义级3D重建 |
| 运动规划 | MoveIt + OMPL | 机器人运动库 |
| 力控 | 阻抗控制 + 力矩传感器 | 精密装配 |
| 边缘运行时 | Docker + ROS2 | 跨平台部署 |

### 5.2 Agent框架

**方案**：层级式Agent架构

- **任务理解Agent**：基于LLM的指令解析
- **任务规划Agent**：基于LLM的任务拆解 + 运动规划
- **技能执行Agent**：基于Skill的原子化执行
- **学习Agent**：基于LoRA的持续微调

## 7. 风险与对策

| 风险 | 影响 | 对策 |
|-----|------|------|
| 语音识别在工业噪声下准确率下降 | 高 | 增加降噪模块 + 关键词优化 |
| 复杂任务规划失败 | 中 | 增加人工确认环节 + 分步执行 |
| 力控装配稳定性不足 | 高 | 增加多传感器冗余 + 异常检测 |
